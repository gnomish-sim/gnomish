Gnomish::Marker

An internal module for encapsulating markers.



makeMarker

Create a new marker list. Markers are created with a 'marker coding' table that encodes how locus states translate into a marker label. The table is simply a hash object and is encoded as follows:

$marker_coding {locus}{locus_state} = 'label';

For example:

    my %marker_coding;
    $marker_coding {3}{AA} = Z;
    $marker_coding {5}{Aa} = W;

NOTE: this exposes the internal data structure of the marker coding (if the table is created incorrectly, it will result in unpredictable behaviour by the script.) We should create new routines to hide this encoding from the end-user interface.

Also, the creation of new markers in a simulation should be done through the Gnomish::Genome::addMarker class method, instead of calls to this method.

Parameters:

  • String: Name

  • MarkerCoding (reference to hash table with marker coding explained above)

  • AlleleSet (the allele set of the genome)


getMarkerCoding

Return the marker label coding for a particular allele state at the specified locus.

Parameters:

  • Int: Locus

  • String: AlleleState

Example:

    my $marker_label = $marker->getMarkerCoding (
                            Locus => 5,
                            AlleleState => "aA" );



getMarkerCodingForLocus
Return a hash table of the marker codings for a given locus.

Parameters:

  • Int: Locus

Example:

    my %marker_coding;
    $marker_coding {3}{aA} = Z;
    $marker_coding {3}{aa} = W;
    $marker_coding {5}{aA} = T;

    my $marker = new Genome::Marker;
    $marker->makeMarker ( Name => "test",
                          MarkerCoding => \%marker_coding,
                          AlleleSet => "a,A" );

    my %locus_coding = $marker->getMarkingCodingForLocus ( Locus => 3 );

    print $locus_coding{aA}, "\n";
    print $locus_coding{aa}, "\n";

Output:

    Z

    W


The returned table looks like:

    $locus_coding{aA} = Z;
    $locus_coding{aa} = W;
