Gnomish::Genome

This class defines the genomic properties of the organism.



makeGenome

Initialize the genome object with provided parameters.

Parameters:

  • String: Name

  • Int: Nloci

  • String: RecombinationDists (comma separated values of type float)

  • String: AlleleSet (comma separated values of of either numbers or letters, 0-9, a-z, A-Z).

Example:

    my $genome = new Gnomish::Genome;
    $genome->makeGenome ( Name => "BarleyHT",
                          Nloci => 5,
                          RecombinationDists => "0.1, 0.1,0.5,0.1,0.1",
                          AlleleSet => "H,T,U,F" );


createUniformGenome

Initialize the genome with uniform recombination distances.

Parameters:

  • Int: NlociPerChromosome

  • Int: NChromosomes

  • Float: RecombinationDist

  • String: AlleleSet (in csv format)

Example:

    my $genome = new Gnomish::Genome;
    $genome->createUniformGenome ( Name => "SimplePlant",
                                   NlociPerChromosome => 5,
                                   NChromosome => 7,
                                   RecombinationDist => 0.1,
                                   AlleleSet => "A,C,K,L" );


createRandomGenome

Initialize genome with a randome RecombinationDist array. (This was created for debugging/testing purposes.)

Parameters:

  • Int: Nloci

  • Int: MaxChromosomeLen

  • String: AlleleSet (in csv format)

Example:

    my $genome = new Gnomish::Genome;
    $genome->createRandomGenome ( Name => "RandomGenome",
                                  MaxChromosomeLen => 10,
                                  Nloci => 56,
                                  AlleleSet => "0,1" );

validAllele

Check and see if the specified allele is valid for the genome. Returns a '1' if the allele is valid, '0' if not.

Parameters:

  • String: Allele

Example:

    if ( $genome->validAllele ( Allele => "A" ) == 1)
  {
        print "Allele is valid\n";
  }
  else
  {
        print "Allele is invalid\n";
  }

addTrait

Add a new trait definition to the Genome.

Parameters:

  • String: Name

  • Float: TraitMean

  • Float:TraitVariance

  • HashTable: TraitAlleleEffects (reference)

Note:

The 'TraitAlleleEffects" table is built like a two dimensional array, with the locus number as the first dimension and the allele state as the second dimension.

Although it is efficient to define the effects of the trait in this way, it is somewhat dangerous. We should add routines to build the table internally to hide it's structure from user scripts. (If the table is built incorrectly, the program could behave in weird and unexpected ways.)

Example:

    my %TraitAlleleEffects;
    $TraitAlleleEffects{717}{HH} = -1.1;
    $TraitAlleleEffects{717}{TT} = 1.1;
    $TraitAlleleEffects{806}{UU} = 0.5;
    $TraitAlleleEffects{806}{FF} = 1.5;
    $TraitAlleleEffects{1190}{FH} = -1.65;
    $TraitAlleleEffects{1190}{FT} = -0.35;

    $genome->addTrait ( Name => "Height",
                        TraitMean => 89;
                        TraitVariance => 5,
                        TraitAlleleEffects => \%TraitAlleleEffects );



addMarker

Add a new marker definition to the genome.

Parameters:

  • String: Name

  • Hashtable: MarkerCoding (refence).

Note:

The 'MarkerCoding' table is built in the same way that the TraitAlleleEffects is built. Hence, the same potential problems apply.

Example:

    my %MarkerCoding;
    $MarkerCoding{1}{HH} = "H";
    $MarkerCoding{1}{TT} = "T";
    $MarkerCoding{21}{HH} = 'H';
    $genome->addMarker ( Name => "DNAmarkers",
                         MarkerCoding => %MarkerCoding );
